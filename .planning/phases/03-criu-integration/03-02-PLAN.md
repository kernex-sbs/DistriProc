# Phase 03 Plan 02: Lazy Pages Integration

## Goal
Integrate CRIU's `lazy-pages` feature with our `page_server.py`. The objective is to restore a process using `criu restore --lazy-pages` and have it fetch missing memory pages from our Python server on demand.

## Context
Phase 2 implemented a custom UFFD handler and TCP server.
Phase 3 aims to apply this to arbitrary processes using CRIU.
CRIU has a built-in `--lazy-pages` mode. Instead of writing a custom C loader (like `test_uffd.c`), we will leverage CRIU's restore capability but backend it with our server.

  - Start `page_server.py` pointing to the dump images.

## Technical Details
- **CRIU Lazy Architecture**:
  - `criu restore --lazy-pages --address <IP> --port <PORT>`
  - This typically expects a `criu-lazy-pages` daemon or a `criu dump` instance on the other side.
- **Challenge**:
  - We need to serve the pages *from the checkpoint files* (or memory) via our Python server.
  - We need to reverse-engineer or implement the CRIU socket protocol.
