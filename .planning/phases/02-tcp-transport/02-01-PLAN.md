# Phase 02 Plan 01: TCP Transport Implementation

## Goal
Implement the core network transport layer for DistriProc, enabling the `userfaultfd` handler to fetch pages from a remote Python server over TCP instead of generating them locally.

## Context
In Phase 1, we proved we can trap page faults and handle them in userspace (`test_uffd.c`).
In Phase 2, we move the data source out of the process address space to a separate process (simulating a remote machine). This is the "Distri" in DistriProc.

## Tasks

- [ ] **1. Create Page Server (`page_server.py`)**
  - Implement a simple TCP server in Python.
  - Listen on a specific port (e.g., 9999).
  - Protocol: Receive 8-byte address (or page index), return 4096 bytes of data.
  - Data source: Generate a pattern (e.g., repeated byte based on address) to verify correctness.

- [ ] **2. Create TCP Client (`src/test_uffd_tcp.c`)**
  - Copy `src/test_uffd.c` to `src/test_uffd_tcp.c`.
  - Add TCP connection logic in `main()` before registering UFFD.
  - Update `fault_handler_thread`:
    - Instead of `memset(0)`, send the faulting address to the server.
    - Read 4096 bytes from the socket.
    - Use `UFFDIO_COPY` to place the received data into memory.

- [ ] **3. Verification**
  - Start `page_server.py`.
  - Run `src/test_uffd_tcp`.
  - Verify that the data in memory matches what the server sent.

## Technical Details

- **Protocol**: Simple request/response.
  - Request: `uint64_t page_index` (8 bytes)
  - Response: `uint8_t data[4096]` (4KB raw page)
- **Error Handling**:
  - Basic socket error handling (exit on failure for PoC).
  - Ensure `recv` loops until full page is received.

## Verification Steps
1. Run `python3 src/page_server.py &`
2. Run `./src/test_uffd_tcp`
3. Expect output: "Fault handled... Value at address: [Correct Value from Server]"
